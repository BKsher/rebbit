--------------------------------------------------------------------------------------

User Management

--------------------------------------------------------------------------------------

Register a User
Endpoint: POST /auth/register
Description: Allows new users to register. Requires username, email, and password.

User Login
Endpoint: POST /auth/login
Description: Authenticates a user and returns a session token or cookie. Requires email and password.

--------------------------------------------------------------------------------------

Thread Management

--------------------------------------------------------------------------------------

Create a Thread
Endpoint: POST /threads
Description: Allows a registered user to create a new thread. Requires thread title and user identification (handled via authentication token).

List All Threads
Endpoint: GET /threads
Description: Retrieves a list of all threads.

Get a Single Thread
Endpoint: GET /threads/:threadId
Description: Retrieves detailed information about a specific thread, including posts. The threadId is a parameter in the URL.

Update a Thread
Endpoint: PUT /threads/:threadId
Description: Allows the author to update a thread's title. Requires updated title and user identification.


Delete a Thread
Endpoint: DELETE /threads/:threadId
Description: Allows the author or an admin to delete a thread.

--------------------------------------------------------------------------------------

Post Management

--------------------------------------------------------------------------------------

Create a Post in a Thread
Endpoint: POST /threads/:threadId/posts
Description: Allows a user to create a post in a specific thread. Requires the content of the post.

Update a Post
Endpoint: PUT /posts/:postId
Description: Allows the author to update a post. Requires updated content.

Delete a Post
Endpoint: DELETE /posts/:postId
Description: Allows the author or an admin to delete a post.

--------------------------------------------------------------------------------------

Comment Management

--------------------------------------------------------------------------------------

Add a Comment to a Post
Endpoint: POST /posts/:postId/comments
Description: Allows users to add comments to a specific post. Requires the content of the comment.

Update a Comment
Endpoint: PUT /comments/:commentId
Description: Allows the author to update a comment. Requires updated content.

Delete a Comment
Endpoint: DELETE /comments/:commentId
Description: Allows the author or an admin to delete a comment.

-------------------------------------------------------------------------------------


Аутентификация: Большинству endpoints, особенно тем, которые создают, обновляют или удаляют данные, потребуется аутентификация пользователя. Обычно это обрабатывается с помощью токена в заголовке запроса.

Данные ответа: каждый endpoint будет возвращать данные в формате JSON вместе с соответствующими кодами состояния HTTP.

Обработка ошибок: в случае ошибки (например, недопустимого ввода или несанкционированного доступа) конечные точки будут возвращать сообщения об ошибках и соответствующие коды состояния HTTP.
